# CachyRS
A third party client for RS3 (and eventually, OSRS.)

**THIS PROJECT IS A WIP AND ACTIVELY UNDER DEVELOPMENT**
## Compiling
### Dependencies
* `Capstone`
### Linux
`git clone https://github.com/csmoke66/CachyRS`  
`cd CachyRS`  
`chmod +x build.sh`  
`./build.sh`  
## Usage
1. Run the native RS3 Linux client
2. Run `bin/injector` with root permissions

## Project Structure
### updater
This is responsible for generating the headers that map to game structures in memory. To generate new headers simply run `bin/updater` with rs2client in the same directory.
### mod
This is the shared library that's injected into `rs2client` to provide all of the client functionality. It relies on headers generated by the updater to interact with the game's memory.
### injector
This is a simple `.so` injector that was modified from another repository I forget the name of.
## Feature Requests
I expect people to request things like Windows support, or language bindings. This is currently outside of the scope of the project as I only run Linux on my machines. I will happily work with people to add support and accept pull requests.  
  
All Linux specific APIs have multi-platform stubs:
```c
#ifdef __linux__
// linux code
#else
UNSUPPORTED_OS();
#endif
```
The C++ STL is used wherever possible to reduce reliance on OS specific APIs. Adding support for another system should just be adding additional macros.
